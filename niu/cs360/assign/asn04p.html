Program 4A -- Table Processing
and Internal Subroutines
(100 points)

Overview

DeKalb Discs keeps information about the CDs it currently has in stock. For each CD, a stock number is assigned and a record that contains information about the CD is created. The records are entered into the database as new CDs arrive. The owner of DeKalb Discs wants to build a database of the CDs currently in stock.

The owner has hired you, a reputable assembler programmer, to accomplish this task. You will build a table of stock information. Each table entry will consist of the stock number for the CD, the artist ID number, the number of copies currently in stock, and the selling price (in cents) of the CD. Each table element will be 4 fullwords in length. Each item will be stored in a fullword as a binary number.

INPUT:

The input for this program is an unknown number of records (maximum of 20) that will be used to build the table. Each record has the following format: a stock number, an artist ID number, the number of copies currently in stock, and the selling price (in cents) of the CD.

OUTPUT:

The output for the program will be a report that lists the current stock information. Use the following report as a model for your report.



                   DEKALB DISCS STOCK INFORMATION


        STOCK NUMBER     ARTIST ID     # IN STOCK     CD PRICE

        ------------     ---------     ----------     --------

       

           123523           1234           20           1299


           511523           8469           46           1599

             

           610543           1794           31           1399

               

           319854           1794           25            999
PROCESSING REQUIREMENTS:
  1. You should use at least 3 internal subroutines: MAIN, BUILD, and PRINT.
    • MAIN will call the other subroutines, passing the appropriate parameters to each. It should print the page and column headings for the report BEFORE calling the PRINT routine. The table should be defined in the storage area for the MAIN routine and it should hold a maximum of 20 elements, each of which is 4 fullwords in length.

    • BUILD will read records from the input file and put the information into the table. INPUT PARAMETERS:
      • Address of the beginning of the table
      • Address of a fullword in which to store the address of the end of the table
      PRINT will print the contents of the table after it has been built

      INPUT PARAMETERS:
      • Address of the beginning of the table
      • Address of the fullword that contains the address of the end of the table
  2. Before exiting the program, use XDUMP to dump the entire table.

  3. The report should begin on a new page and should be double spaced.

  4. Test the program with your own instream data. For the final run, replace your instream data with the following JCL:
    //FT05F001 DD DSN=T90MES1.C360(DATA4),DISP=SHR
