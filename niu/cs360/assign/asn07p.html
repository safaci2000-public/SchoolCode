Program 7 - Table Processing,
Decimal Arithmetic, ED/EDMK
(150 Points)

Overview

This assignment continues development of the programming support for the "CD Empire".  This program must combine several CD inventories and, following the addition of each new inventory to the data base, you must sort and print the data base as it stands at that time.  Your program will not be allowed to make any use of XDECI or XDECO.  All numeric data will be stored in the table in Packed Decimal format and all mathematical computations involving this data will use Decimal Arithmetic.  For data input you will PACK the character (EBCDIC) input into Packed Decimal format.  Use ED to format the packed fields except for price. Use EDMK to format the price field so that a dollar sign can be inserted before the first digit in the field.

INPUT:
--------

     There will be several sets of "inventory" and one set of transactions, all contained in a single file.

1)  The first several sets of inventory data each contain an unknown number of records.  (The total number of records in the combined data base will be no more than 25).  These records will contain additional information (an ISBN number) and they will be in a different format, as follows:

                 Columns               Description
                 ---------               -----------
                      1-6                  Stock Number                      (numeric data)
                      7                     Blank
                      8-23               Artist                                     (character data)
                     24                    Blank
                     25-47              Title                                      (character data)
                     48                    Blank
                     49-51              Number of Copies in Stock   (numeric data)
                     52                    Blank
                     53-57              Price     (cents)                      (numeric data)
                     58                    Blank
                     59-67              ISBN-Partial                         (numeric data)

      The end of each of the "inventories" is marked by a record with a stock number of 'xxxx0f', where the xxxx will be either character zero or blank, 0 will be a character zero, and f will be a flag that when set to C'T' will indicate that this is the end of the last of the "inventories" and the records that follow will be transaction records.  Do NOT put these "zeros" records into the table.  When this record is encountered in BUILD, you will return from BUILD to the MAIN routine.

      The last set of data consists of an unknown number of transaction records.  Each contains the following information:

                 Columns               Description
                 ---------               -----------
                      1-23               Title                 (character data)
                    24-27               Blanks
                    28-30               Number of CDs  (Returned or Sold)  (numeric data)
                    31-34               Blanks
                    35                    R or S (Return or Sale)  (character data)

OUTPUT:
----------
       The output from this program will consist of one report for each of the groups of "inventory" data.  Your MAIN program should have a loop that calls BUILD, SORT, and PRINT in sequence.  After return from PRINT, you should check the INPUT-BUFFER to see if Card Column 6 of the last card read was a C'T'.  If it was not, you should return to the top of the loop and perform another BUILD, SORT, and PRINT.  When your test in the MAIN program on return from PRINT shows C'T' in Card Column 6, exit the loop and proceed to the UPDATE subroutine to perform the transaction processing.  The following is a sample report produced by the PRINT subroutine.

                                    SORTED CD STOCK FOR "CD EMPIRE"

STOCK#   ARTIST                       TITLE                         QUANTITY    PRICE               ISBN
                                                                                           ON HAND
-----------------------------------------------------------------------------------------------------
192766      MONICA                   WHITE OUT                      20              $15.82       123456879
653382      CREED                       HUMAN CLAY                 30              $10.95       123987789

      At the end of each PRINT subroutine, print the following information:
               Total inventory value, that is sum of (number)*(price)
               Total number of CDs in the inventory.
               Average CD value (inventory value/number of CDs - and rounded with SRP)

      Your processing of the transaction records will produce a transaction report.  There should be a detail line in the report for each CD title.  Your transaction report will include the values shown in this sample transaction report:

                                TRANSACTION REPORT FOR "CD EMPIRE"

   ARTIST                     TITLE                 QUANTITY   RETURN   AMOUNT   QUANTITY
                                                                 ON HAND        OR                              ON HAND
                                                                 BEFORE         SALE                             AFTER
-------------------------------------------------------------------------------------------------
CREED                      HUMAN CLAY          30              SALE             10                20
SOLAR ECLIPSE     LA SUNRISE              15            RETURN            3                18

      At the end of the transaction report, provide a statement of:
               Total change in inventory value, that is sum of (number)*(price)
                        (followed by minus sign if value decreased)
               Total change in number of CDs in the inventory
                        (followed by minus sign if number decreased)

      Your final report will be another run of the PRINT subroutine.  It is Required that the value of the CD Inventory (given at the end of Each PRINT subroutine) correspond with the Change In Inventory Value given at the end of the Transaction Report produced by the UPDATE subroutine.

TABLE  REQUIREMENTS:
-----------------------------

Each entry in the table should now have the following format:

      SIZE                             STORED
 ------------                      -------------------
 PL4                                  Stock Number
 CL16                               Artist Name
 PL5                                  ISBN
 CL23                               Title
 PL3                                  Price:   DDDCC   D=Dollars, C=Cents
 PL2                                  Quantity on Hand
 XL11                                     Unused

Requirements

1. The input parameters to BUILD, SORT,  PRINT, and UPDATE are the same as they were in Assignment # 6.  Prior to return, BUILD always updates the ANA.  Title and column headers for the reports should be printed in the MAIN routine before calling PRINT or UPDATE.  Summary values at the end of reports produced by PRINT and UPDATE are printed from within those subroutines.

2. This assignment will use one table with 25 entries.  Each entry is 64 bytes in length (as it was in Assignment # 6).  In the MAIN routine include:

                               ORG   MAIN+((*-MAIN+31)/32*32)
                               DC      CL28'THE ANA IS ON THE LEFT OF THIS'
              ANA         DC     A(TAB)
                                DC    CL32' THE TABLE BEGINS HERE'
              TAB          DC     F'-1'

3. The ANA, the Table, and the Input Buffer are again located in the MAIN routine. Any subroutine that has an XREAD will need the address of the buffer.

4. The PRINT and UPDATE subroutines will use the assembler instructions, ED and EDMK to output the Packed Decimal numbers.

5. All calculations are done with the packed decimal instructions.

6. Use the SRP instruction ("i"=5) to round the average you calculate in the PRINT routine to the nearest cent. Make sure you define you packed fields large enough to allow for extra decimal places before the division, so that there will be significant digits after the hundredth place. Otherwise there are no additional digits to round.  (Remember this is still integer division with a quotient and a remainder.)

7. Each output section should have a header and start on a new page.

8. The program must end normally unless directed otherwise by your instructor.

9. The input data will be found in the following file: //FT05F001 DD DSN=T90MES1.C360(DATA7),DISP=SHR
