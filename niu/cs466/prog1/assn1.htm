In this assignment you will be creating an IMS database which manages a hotel reservation system.  This database would be used to track the reservations in various hotel locations.

 

The fields of each segment type in the database are given below. A "K" denotes a key field.  A U indicates a unique key and an N indicates a non-unique key.  Assume that all the other fields could be potential search fields and must be defined by name. 

Note: All dates are in the format: YYYYMMDD.

1.  Code and assemble the DBD for the database.

 

2.  Code and assemble a PSB for loading the database. 

 

3.  Write a COBOL program to load the database.

 

4.  Use IMSSEG85 to link edit the DBD, the two PSB's and load program and then execute the load program.  Also provide a DD card to dump the database into a sequential dataset.

 

5.  Use the DFSDUMP utility to list the dumped database.

 

6.  The data for loading the database is located in:

T90MBM1.HOTEL.DATA1.

The format of the card is:

  col 1:       segment type:  1 = HOTEL            4 = ROOMS  

                              2 = RATES            5 = CUSTOMER

                              3 = DAYS             6 = CHARGES

 

  col 2 - 80:  the data for the rest of the segment.  The format depends upon the type and list listed above.

 

Do not assume that the segments are in the proper order.  Check the hierarchical sequence and if any of the segments are out of order or you encounter an invalid segment type print an error message that contains the segment contents, the status code and an error note.  This should be done by testing the status code for errors.  Remember each IMS call returns a status code and it must be checked after each call.  Please let us know if you find any segments out of order so that we can fix the data set!

 

 

Your program must count the number of occurrences of each segment type inserted plus the total number of segments in the database.  After all segments have been loaded print the count information.

 

Note:

 

Make the first step of your program an IEFBR14 step that deletes the data set containing the database.  This way each time you run the program it will delete the database and then you can make the DISP equal to NEW when you create the database.

 

Put the database files in a permanent PDS, you will need it for assignments 2 and 3.  Save this program because if anything happens to your database you can rerun it and recreate the database.

 

CODE DESCRIPTIONS BY SEGMENT TYPE:

 

HOTEL:

The following fields will either have a value of 'Y' or 'N'.  Y means that option is available and N means that option is not available.

     POOL       PIC X

     PETS       PIC X

     FOOD       PIC X

     LAUNDRY    PIC X

     WEIGHTRM   PIC X

 

RATES:

     TYPEROOM  PIC X     

          S = SINGLE

          D = DOUBLE

          K = KING

 

     TYPEWEEK  PIC X

          D = WEEKDAY (SUNDAY THRU THURSDAY)

          W = WEEKEND (FRIDAY AND SATURDAY)

    

DAYS:

     WEEKTYPE  PIC X

          D = WEEKDAY (SUNDAY THRU THURSDAY)

          W = WEEKEND (FRIDAY AND SATURDAY)

 

ROOMS:

     ROOMTYPE  PIC X

          S = SINGLE

          D = DOUBLE

          K = KING

 

     SMOKING   PIC X

          S = SMOKING

          N = NON-SMOKING

 

     BOOKED    PIC X

          Y = BOOKED

          N = AVAILABLE

         

     GUARANTE  PIC X

          Y = GUARANTEED LATE

          N = NOT CUARANTEED

 

     FILLED    PIC X

          Y = OCCUPIED

          N = EMPTY

 

CUSTOMER:

     DISCTYPE  PIC X

          S = SENIOR

          C = HOTEL CLUB

          B = BOTH SENIOR AND HOTEL CLUB

 

 

CHARGES:

     CHRGTYPE  PIC  X   

    

          F = FOOD

          T = TELEPHONE

          R = ROOM
